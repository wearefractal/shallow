00068 // CPU Registers.
00069 //
00070 // 1) We would prefer to use an enum, but enum values are assignment-
00071 // compatible with int, which has caused code-generation bugs.
00072 //
00073 // 2) We would prefer to use a class instead of a struct but we don't like
00074 // the register initialization to depend on the particular initialization
00075 // order (which appears to be different on OS X, Linux, and Windows for the
00076 // installed versions of C++ we tried). Using a struct permits C-style
00077 // "initialization". Also, the Register objects cannot be const as this
00078 // forces initialization stubs in MSVC, making us dependent on initialization
00079 // order.
00080 //
00081 // 3) By not using an enum, we are possibly preventing the compiler from
00082 // doing certain constant folds, which may significantly reduce the
00083 // code generated for some assembly instructions (because they boil down
00084 // to a few constants). If this is a problem, we could change the code
00085 // such that we use an enum in optimized mode, and the struct in debug
00086 // mode. This way we get the compile-time error checking in debug mode
00087 // and best performance in optimized code.
00088 //
