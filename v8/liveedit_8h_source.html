00033 // Live Edit feature implementation.
00034 // User should be able to change script on already running VM. This feature
00035 // matches hot swap features in other frameworks.
00036 //
00037 // The basic use-case is when user spots some mistake in function body
00038 // from debugger and wishes to change the algorithm without restart.
00039 //
00040 // A single change always has a form of a simple replacement (in pseudo-code):
00041 //   script.source[positions, positions+length] = new_string;
00042 // Implementation first determines, which function's body includes this
00043 // change area. Then both old and new versions of script are fully compiled
00044 // in order to analyze, whether the function changed its outer scope
00045 // expectations (or number of parameters). If it didn't, function's code is
00046 // patched with a newly compiled code. If it did change, enclosing function
00047 // gets patched. All inner functions are left untouched, whatever happened
00048 // to them in a new script version. However, new version of code will
00049 // instantiate newly compiled functions.
