00517 // ES6 Draft Rev3 10.2 specifies declarative environment records with mutable
00518 // and immutable bindings that can be in two states: initialized and
00519 // uninitialized. In ES5 only immutable bindings have these two states. When
00520 // accessing a binding, it needs to be checked for initialization. However in
00521 // the following cases the binding is initialized immediately after creation
00522 // so the initialization check can always be skipped:
00523 // 1. Var declared local variables.
00524 //      var foo;
00525 // 2. A local variable introduced by a function declaration.
00526 //      function foo() {}
00527 // 3. Parameters
00528 //      function x(foo) {}
00529 // 4. Catch bound variables.
00530 //      try {} catch (foo) {}
00531 // 6. Function variables of named function expressions.
00532 //      var x = function foo() {}
00533 // 7. Implicit binding of 'this'.
00534 // 8. Implicit binding of 'arguments' in functions.
00535 //
00536 // ES5 specified object environment records which are introduced by ES elements
00537 // such as Program and WithStatement that associate identifier bindings with the
00538 // properties of some object. In the specification only mutable bindings exist
00539 // (which may be non-writable) and have no distinct initialization step. However
00540 // V8 allows const declarations in global code with distinct creation and
00541 // initialization steps which are represented by non-writable properties in the
00542 // global object. As a result also these bindings need to be checked for
00543 // initialization.
00544 //
00545 // The following enum specifies a flag that indicates if the binding needs a
00546 // distinct initialization step (kNeedsInitialization) or if the binding is
00547 // immediately initialized upon creation (kCreatedInitialized).
00548 enum InitializationFlag {
00549   kNeedsInitialization,
00550   kCreatedInitialized
00551 };
