00063   // This is used by the heap traversal to enter the addresses into the store
00064   // buffer that should still be in the store buffer after GC.  It enters
00065   // addresses directly into the old buffer because the GC starts by wiping the
00066   // old buffer and thereafter only visits each cell once so there is no need
00067   // to attempt to remove any dupes.  During the first part of a GC we
00068   // are using the store buffer to access the old spaces and at the same time
00069   // we are rebuilding the store buffer using this function.  There is, however
00070   // no issue of overwriting the buffer we are iterating over, because this
00071   // stage of the scavenge can only reduce the number of addresses in the store
00072   // buffer (some objects are promoted so pointers to them do not need to be in
00073   // the store buffer).  The later parts of the GC scan the pages that are
00074   // exempt from the store buffer and process the promotion queue.  These steps
00075   // can overflow this buffer.  We check for this and on overflow we call the
00076   // callback set up with the StoreBufferRebuildScope object.
00077   inline void EnterDirectlyIntoStoreBuffer(Address addr);
00078 
00079   // Iterates over all pointers that go from old space to new space.  It will
00080   // delete the store buffer as it starts so the callback should reenter
00081   // surviving old-to-new pointers into the store buffer to rebuild it.
00082   void IteratePointersToNewSpace(ObjectSlotCallback callback);
