00036 // Maintains a per-v8thread stack area that can be used by irregexp
00037 // implementation for its backtracking stack.
00038 // Since there is only one stack area, the Irregexp implementation is not
00039 // re-entrant. I.e., no regular expressions may be executed in the same thread
00040 // during a preempted Irregexp execution.

00042  public:
00043   // Create and delete an instance to control the life-time of a growing stack.
