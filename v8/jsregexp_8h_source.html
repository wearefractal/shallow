01101 // Improve the speed that we scan for an initial point where a non-anchored
01102 // regexp can match by using a Boyer-Moore-like table. This is done by
01103 // identifying non-greedy non-capturing loops in the nodes that eat any
01104 // character one at a time.  For example in the middle of the regexp
01105 // /foo[\s\S]*?bar/ we find such a loop.  There is also such a loop implicitly
01106 // inserted at the start of any non-anchored regexp.
01107 //
01108 // When we have found such a loop we look ahead in the nodes to find the set of
01109 // characters that can come at given distances. For example for the regexp
01110 // /.?foo/ we know that there are at least 3 characters ahead of us, and the
01111 // sets of characters that can occur are [any, [f, o], [o]]. We find a range in
01112 // the lookahead info where the set of characters is reasonably constrained. In
01113 // our example this is from index 1 to 2 (0 is not constrained). We can now
01114 // look 3 characters ahead and if we don't find one of [f, o] (the union of
01115 // [f, o] and [o]) then we can skip forwards by the range size (in this case 2).
01116 //
01117 // For Unicode input strings we do the same, but modulo 128.
01118 //
01119 // We also look at the first string fed to the regexp and use that to get a hint
01120 // of the character frequencies in the inputs. This affects the assessment of
01121 // whether the set of characters is 'reasonably constrained'.
01122 //
01123 // We also have another lookahead mechanism (called quick check in the code),
01124 // which uses a wide load of multiple characters followed by a mask and compare
01125 // to determine whether a match is possible at this point.
01126 enum ContainedInLattice {
01127   kNotYet = 0,
01128   kLatticeIn = 1,
01129   kLatticeOut = 2,
01130   kLatticeUnknown = 3  // Can also mean both in and out.
01131 };

01134 inline ContainedInLattice Combine(ContainedInLattice a, ContainedInLattice b) {
01135   return static_cast<ContainedInLattice>(a | b);
01136 }
