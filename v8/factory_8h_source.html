00093   // String creation functions.  Most of the string creation functions take
00094   // a Heap::PretenureFlag argument to optionally request that they be
00095   // allocated in the old generation.  The pretenure flag defaults to
00096   // DONT_TENURE.
00097   //
00098   // Creates a new String object.  There are two String encodings: ASCII and
00099   // two byte.  One should choose between the three string factory functions
00100   // based on the encoding of the string buffer that the string is
00101   // initialized from.
00102   //   - ...FromAscii initializes the string from a buffer that is ASCII
00103   //     encoded (it does not check that the buffer is ASCII encoded) and
00104   //     the result will be ASCII encoded.
00105   //   - ...FromUtf8 initializes the string from a buffer that is UTF-8
00106   //     encoded.  If the characters are all single-byte characters, the
00107   //     result will be ASCII encoded, otherwise it will converted to two
00108   //     byte.
00109   //   - ...FromTwoByte initializes the string from a buffer that is two
00110   //     byte encoded.  If the characters are all single-byte characters,
00111   //     the result will be converted to ASCII, otherwise it will be left as
00112   //     two byte.
00113   //
00114   // ASCII strings are pretenured when used as keys in the SourceCodeCache.
00115   Handle<String> NewStringFromAscii(
00116       Vector<const char> str,
00117       PretenureFlag pretenure = NOT_TENURED);
