00154 // Support for alternative bool type. This is only enabled if the code is
00155 // compiled with USE_MYBOOL defined. This catches some nasty type bugs.
00156 // For instance, 'bool b = "false";' results in b == true! This is a hidden
00157 // source of bugs.
00158 // However, redefining the bool type does have some negative impact on some
00159 // platforms. It gives rise to compiler warnings (i.e. with
00160 // MSVC) in the API header files when mixing code that uses the standard
00161 // bool with code that uses the redefined version.
00162 // This does not actually belong in the platform code, but needs to be
00163 // defined here because the platform code uses bool, and platform.h is
00164 // include very early in the main include file.

00281 // The expression OFFSET_OF(type, field) computes the byte-offset
00282 // of the specified field relative to the containing type. This
00283 // corresponds to 'offsetof' (in stddef.h), except that it doesn't
00284 // use 0 or NULL, which causes a problem with the compiler warnings
00285 // we have enabled (which is also why 'offsetof' doesn't seem to work).
00286 // Here we simply use the non-zero value 4, which seems to work.
00287 #define OFFSET_OF(type, field)                                          \
00288   (reinterpret_cast<intptr_t>(&(reinterpret_cast<type*>(4)->field)) - 4)

00370 // The different language modes that V8 implements. ES5 defines two language
00371 // modes: an unrestricted mode respectively a strict mode which are indicated by
00372 // CLASSIC_MODE respectively STRICT_MODE in the enum. The harmony spec drafts
00373 // for the next ES standard specify a new third mode which is called 'extended
00374 // mode'. The extended mode is only available if the harmony flag is set. It is
00375 // based on the 'strict mode' and adds new functionality to it. This means that
00376 // most of the semantics of these two modes coincide.
00377 //
00378 // In the current draft the term 'base code' is used to refer to code that is
00379 // neither in strict nor extended mode. However, the more distinguishing term
00380 // 'classic mode' is used in V8 instead to avoid mix-ups.
00381 
00382 enum LanguageMode {
00383   CLASSIC_MODE,
00384   STRICT_MODE,
00385   EXTENDED_MODE
00386 };
