00038 // Maximum number of significant digits in decimal representation.
00039 // The longest possible double in decimal representation is
00040 // (2^53 - 1) * 2 ^ -1074 that is (2 ^ 53 - 1) * 5 ^ 1074 / 10 ^ 1074
00041 // (768 digits). If we parse a number whose first digits are equal to a
00042 // mean of 2 adjacent doubles (that could have up to 769 digits) the result
00043 // must be rounded to the bigger one unless the tail consists of zeros, so
00044 // we don't need to preserve all the digits.
00045 const int kMaxSignificantDigits = 772;

00060 // The fast double-to-(unsigned-)int conversion routine does not guarantee
00061 // rounding towards zero.
00062 // The result is unspecified if x is infinite or NaN, or if the rounded
00063 // integer value is outside the range of type int.
00064 inline int FastD2I(double x) {
00065   // The static_cast convertion from double to int used to be slow, but
00066   // as new benchmarks show, now it is much faster than lrint().
00067   return static_cast<int>(x);
